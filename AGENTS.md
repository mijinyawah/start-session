## Architecture

Plugin format: distributable `.plugin` zip. Also installable as a file clone (scaffold/ folder method).

**Key directories:**

| Path | Purpose |
|------|---------|
| `commands/` | 6 slash commands: `/setup`, `/start-session`, `/end-session`, `/checkpoint`, `/new-project`, `/migrate` |
| `skills/session-management/` | SKILL.md context layer — auto-loaded on session-related topics |
| `scaffold/` | Workspace template files generated by `/setup` into the user's working directory |
| `.claude-plugin/` | `plugin.json` (manifest) + `marketplace.json` (install registry) |

**Logic/data split:**
- Session protocols and command behavior → live in the plugin → update automatically with each push
- User identity, preferences, project state → live in workspace files → never touched by plugin updates

**Update model:** Commit-SHA versioning. No `version` field in `plugin.json` — every push to `main` is a new version. Users get updates automatically at next Claude startup.

**Install commands:**
```
/plugin marketplace add mijinyawah/start-session
/plugin install start-session@start-session
```
